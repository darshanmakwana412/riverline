# Handoff: Q4, Q5 Detection and C3 DNC Fix

**Date:** 2026-04-23
**Session focus:** Implemented the Q4 (remembering context) and Q5 (no repetition) quality checks from spec section 9, and fixed two gaps in the C3 (DNC compliance) check identified in `docs/validations/2026-04-22-003-state-transition-invariant-audit.md` (GAP-2 and GAP-3). All changes in `eval_takehome.py`.

---

## What Was Accomplished

### C3 DNC fix

Two issues in the existing `_check_compliance`:

1. `DNC_RE` matched English phrases only. A scout across `data/production_logs.jsonl` found 68 conversations where the borrower used Hindi or Hinglish DNC phrasing (for example `band karo`, `phone mat karna`, `pareshaan mat kar`, `tang karna band karo`, `baar baar message karna band karo`) which the English regex missed entirely.
2. The check emitted one `C3_dnc_violation` per post-DNC bot message. In the same shared-turn data-quality pattern that inflated Q2, this produced many violations for what is a single regulatory failure. Post-escalation re-contact conversations (`032101dc`, `c1cffa15`, `91faad79`, `18047863`, `7d118cdb`) each produced multiple C3 rows.

**Fix:** Extended `DNC_RE` with Hindi and Hinglish alternations. Rewrote the post-DNC loop to collect the list of offending bot turns and emit a single `C3_dnc_violation` at the first offending turn. Severity scales with count using `min(1.0, 0.7 + 0.1 * (count - 1))`. The explanation string includes the DNC turn, the first offending bot turn, and the total count of post-DNC bot messages. C5 (threats) detection is unchanged and still fires per-message.

### Q5 repetition check

New function `_check_q5_repetition(messages)` plus helper `_normalize_bot_text`. Normalization strips emoji ranges, collapses non-word characters to single spaces, and lowercases. Bot messages with normalized text shorter than 20 characters are skipped to avoid flagging short acknowledgements like `ok` or `theek hai`. Messages are grouped by normalized text; any group with at least two members produces one `Q5_repetition` violation at the second occurrence's turn. Severity is `min(0.9, 0.3 + 0.2 * (count - 1))`. The explanation lists all turns in the group.

### Q4 context check

New function `_check_q4_context(messages, transitions)` covering two sub-rules:

- `Q4_reintroduction`: bot message at turn greater than zero matching `INTRO_RE` (phrases like `this is priya`, `my name is priya`, `calling from riverline`, `from riverline financial`). Severity 0.7.
- `Q4_reverification`: bot message matching `VERIFY_RE` (verification keywords including `verify`, `date of birth`, `last 4 digits`, `pan card`, `aadhaar`, `registered mobile`) at a turn strictly after the first `verification to intent_asked` transition. Severity 0.7.

Each sub-rule is deduplicated to at most one violation per conversation via a flag, so a repeatedly re-introducing bot does not fire Q4 multiple times (Q5 handles the count aspect separately).

### Wire-in

`AgentEvaluator.evaluate` now appends `_check_q4_context(messages, transitions)` and `_check_q5_repetition(messages)` to the violations list.

## Key Decisions

- **C3 dedup is count-weighted, not silenced.** A conversation with many post-DNC bot messages is regulatorily worse than one with a single re-contact. The `0.7 + 0.1 * (count - 1)` scaling reaches 1.0 at four or more post-DNC messages and preserves this gradient without emitting one row per message.
- **Hindi patterns are added to `DNC_RE` rather than a second regex.** Keeps the compliance check as a single pass and keeps the search for the first DNC turn simple. The alternation is organized into English keywords, object-plus-verb pairs (`phone|message|call|sms|contact` with `mat` or `band`), `band karo` variants, cease-harassment phrases, and a bounded `baar baar ... band` pattern.
- **Q5 normalization is aggressive, not fuzzy.** Jaccard or difflib ratio matching would catch more near-duplicates but also introduce tuning risk. Stripping non-word characters and lowercasing catches the primary failure mode (verbatim intro repeats, verbatim amount-pending reminders) at low implementation and evaluation cost. If the hidden test set has near-but-not-identical repetition, this can be revisited.
- **Q5 short-text filter at 20 characters.** Short bot messages (`ok`, `sure`, `theek hai`, acknowledgements) legitimately repeat. Twenty characters is the empirical threshold below which almost all matches are these benign acknowledgements in the production logs.
- **Q4 uses explicit keyword regexes, not semantic inference.** The audit doc flagged concrete patterns (re-introduction, re-verification). Keyword checks catch exactly those patterns without pulling in the ambiguity of general context tracking. Expanding Q4 to cover forgotten prior commitments would require conversation-level state modelling which is out of scope.
- **Q4 is deduped per sub-rule.** One `Q4_reintroduction` and at most one `Q4_reverification` per conversation. Severity does not scale with count because Q5 already captures the repetition signal for the re-introduction case.
- **`_clamp_sev` was intentionally not used.** The existing helper clamps to `[0.3, 0.4]` which would cap all new rules at 0.4. Direct `min(...)` is used. This helper may be a latent bug in the amount-text checks; not addressed in this session.

### Remaining open gaps from the audit doc

- **GAP-1:** Semantic verification bypass. The bot transitions `verification to intent_asked` before the borrower actually confirms identity. Requires an NLP-based check on the preceding borrower message, not just a keyword regex. Not addressed.
- **GAP-4:** Hardship-response sequencing. Bot pushes payment in the same message as empathy. Requires detecting hardship classification at turn T and checking whether the bot at turn T or T+1 contains payment-push language. Not addressed.
